# /ops-doc-draft
> Revisit: when delivery stage structure or document types change. · Last touched: 2026-07-07

## Purpose
Turn raw delivery inputs (notes, transcripts, build/session outputs) into
one of four structured, ready-to-review documents — without inventing
facts or crossing client isolation.

## Inputs
- Source material: session notes, transcripts, build receipts/artifacts,
  or a stated document type request.
- Client/engagement entity page where the document concerns a specific
  client (required for client-ready summaries — refuse without it).
- client_data_boundaries.md rules (binding).

## Steps
1. **Identify document type** — SOP (repeatable process), implementation
   plan (scoped build steps + timeline placeholders), client-ready
   summary (client-facing outcome/status), or founder ops summary
   (internal, Liam-facing). If ambiguous, ask which one rather than
   guessing.
2. **Ground** — pull only from the supplied source material; no invented
   steps, numbers, or claims.
3. **Structure** — match the type's shape:
   - SOP: purpose → trigger → steps → roles → exceptions.
   - Implementation plan: current state → target state → steps →
     dependencies → timeline placeholder → approval gates.
   - Client-ready summary: what was done → outcome → next step (no
     guaranteed-ROI language).
   - Founder ops summary: what happened → decisions needed →
     risks/blockers → next action.
4. **Isolate** — client-specific content stays in that client's folder;
   nothing cross-client blended.
5. **Output** — artifact + receipt with token block. Draft only — never
   sends, publishes, or updates a CRM/hub on its own.

## Never
- Invent steps, numbers, or outcomes not present in the source material.
- Blend multiple clients' details into one document.
- Send, publish, or post the output anywhere.
- Guess the document type when the source doesn't make it clear — ask.

## Done when
One structured document exists, type-matched to its shape, every claim
traceable to source material, client isolation intact. Receipt written.
