# /memory-promotion
> Revisit: when MEMORY_PROMOTION_POLICY.md or the Brain's memory areas change. · Last touched: 2026-07-07

## Purpose
Move durable, verified knowledge into the Business Brain without contaminating it. The Brain stays stable because this gate is strict.

## Inputs
- The candidate: receipt, decision, approved positioning/offer change, or case-note lesson.
- MEMORY_PROMOTION_POLICY.md (binding — the what-gets-promoted / never-promoted lists).

## Steps
1. **Classify** — target destination: decisions/DECISIONS.md · memory/positioning.md or offers.md · relevant memory file · precedent receipt. No clear destination → not ready; return to sender with why.
2. **Verify** — every claim sourced; Liam has reviewed the underlying result. Unreviewed → queue a review question instead of promoting.
3. **Screen** — hard-reject: drafts, raw transcripts, failed runs, old-vault/ZPC/legacy_harvest content, secrets/connector metadata/OAuth state, speculative or unsourced numbers.
4. **Write** — the durable version only (not every draft iteration), by reference where possible, with date + source. Update the target file's Last touched stamp.
5. **Receipt** — record what was promoted, where, and why; token block included.

## Never
- Promote anything on the policy's never-promoted list.
- Copy wholesale when a pointer suffices.
- Promote without Liam review of the underlying result.

## Done when
Candidate is either promoted (target updated, stamped, sourced) or explicitly returned with a one-line reason. Receipt written.
