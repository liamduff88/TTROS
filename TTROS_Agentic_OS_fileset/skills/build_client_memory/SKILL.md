# /build-client-memory — v0 (pre-seeded — hardened by first 3 client uses)
> Revisit: after each of the first 3 client uses; demotion candidate if unused 90 days. · Last touched: 2026-07-07

## Purpose
Consolidate a client's scattered knowledge (Drive, email, meetings, SOPs, Slack, CRM fields, team memory) into a source-backed knowledge base their AI tools can safely use. Mirrors the TTROS substrate — TTR sells what TTROS runs on.

## Playbook (common skeleton, Blueprint V2 §6.2)
- **Stage 0 INTAKE** — entity page + signed scope; isolation folder; data-sensitivity classification of each source category before touching anything.
- **Stage 1 MAP** — source inventory: every place knowledge lives, owner, format, freshness, sensitivity, access method. One-page map, client-approved.
- **Stage 2 DESIGN** — consolidation plan: target structure (entities, processes, offers, FAQs), the **citation rule — "no claim without a source link"**, update/ownership model, what stays out (sensitive/legal/HR as scoped), and every approval gate. Client sign-off.
- **Stage 3 BUILD** — consolidate per plan; every page carries source links; read/draft-only against client systems unless approved_external_action.
- **Stage 4 TEST** — acceptance: spot-check sample claims trace to sources; a defined set of real business questions answered correctly from the base alone. Evidence recorded.
- **Stage 5 HANDOFF** — docs: structure, how to update it, the citation rule as an operating rule, how their AI tools consume it, pause/rollback.
- **Stage 6 CLOSE** — entity page · receipt with token breakdown · access closeout, including revoking TTR access to client sources.

## Never
- Ingest sources outside the scoped, sensitivity-classified inventory.
- Write a claim without a source link.
- Copy client knowledge into TTROS's own substrate or another client's space.
- Retain access to client data stores after Stage 6.

## Done when
Knowledge base delivered matching the approved plan; citation rule verified by spot-check; question-set acceptance passed with evidence; access revoked; receipt written. v0 tracking in queue/skill_trust.jsonl.
