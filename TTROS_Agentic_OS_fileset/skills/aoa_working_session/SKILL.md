# /aoa-working-session
> Revisit: if the AOA framework or session format changes. · Last touched: 2026-07-07

## Purpose
Make each 45-minute session count: one workflow, run through Audit → Optimize → Automate, with artifacts the client can act on.

## The framework (TTR standard)
- **Audit** — map the current workflow: inputs, outputs, tools, delays, handoffs, data sensitivity, owner.
- **Optimize** — simplify before adding AI: remove duplicate steps, clarify roles, define approval points.
- **Automate** — add AI/automation only where useful: off-the-shelf tools, Zapier/Make, GHL, CRM workflows, AI assistants, or a scoped custom agent.

## Steps — prep
1. Pick the session's one workflow from the client intake form + hub open items; confirm with Liam if ambiguous.
2. Draft session agenda: current audit knowledge, questions to close gaps, candidate optimizations, candidate automations with rough effort.
3. Flag data-sensitivity and approval-point questions to raise live.

## Steps — follow-up (within 24h)
4. Write call notes: audit map, optimizations agreed, automations chosen, explicit approval gates.
5. Extract top-3 takeaways, action items (owner + date), decisions.
6. Update the client hub (`/client-hub`) and build manifest; if an automation grew into build scope, log it as "next recommended build" — don't build it here.
7. Receipt with token count.

## Never
- More than one workflow per session.
- Automate before Optimize — no AI bolted onto a broken process.
- Commit to a build price/scope; that's Liam's call.
- Skip defining human approval points on any automated step.

## Done when
Agenda existed before the session; notes, takeaways, actions, and hub update exist within 24h after; receipt written.
