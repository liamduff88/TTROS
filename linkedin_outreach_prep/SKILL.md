# /linkedin-outreach-prep
> Revisit: when ICP, offers, or the LinkedIn lane split changes. · Last touched: 2026-07-07

## Purpose
Give Liam ready-to-send LinkedIn messages grounded in real prospect signals. Lane split rule: marketing owns LinkedIn content; **revenue owns relationship outreach** — this skill is revenue's side. The LinkedIn connector is an access path, not an agent.

## Inputs
- Prospect name(s) from the queue item — sourced from a lead-gen run report or Liam directly. Refuse unnamed "find people" requests (that's /lead-gen-run).
- Prospect entity page if one exists; memory/ideal_clients.md, offers.md.

## Steps
1. **Verify** — confirm the prospect matches ICP and any lead-gen evidence still holds (public signals only).
2. **Angle** — one relevance hook per prospect tied to an observable signal (their post, business change, visible workflow gap). No fabricated familiarity.
3. **Draft** — connection note (if not connected) + first message, in Liam's voice: short, specific, no pitch-dump, one clear low-friction next step (usually the free System Fit Call).
4. **CASL/compliance check** — commercial-message posture respected; no misleading subject or false pretext.
5. **Output** — per-prospect drafts as one artifact; update prospect entity pages with sourced facts only; receipt with token block.

## Never
- Send messages, connection requests, or reactions.
- Invent shared history, urgency, or interest signals.
- Draft for prospects with no verifiable evidence trail.

## Done when
Artifact contains, per prospect: evidence line, angle, connection note + message draft, suggested next step. Receipt written.
