# /content-draft
> Revisit: when marketing_voice or positioning changes. · Last touched: 2026-07-15

## Purpose
Produce one publish-ready draft in Liam's voice. Draft-only; publication is Liam's decision, always.

## Inputs
- Content brief or topic from the queue item (one piece per item).
- `business_brain:memory/marketing_voice.md` + `business_brain:memory/positioning.md` (read both, every time).
- Offer/pricing facts only from `business_brain:memory/offers.md` and `business_brain:memory/sales_and_revenue.md` — never invented.

## Steps
1. **Frame** — restate the piece's job in one line: audience, the one leak/lever it addresses (effectiveness / efficiency / quality), desired next step.
2. **Answer-first block** — open with the direct answer to the question the piece targets (AEO), then expand.
3. **Draft** — in marketing_voice: plain, operator-level, no hype, no guaranteed-outcome claims (offers.md "do not claim" list is binding).
4. **Voice check** — reread against marketing_voice.md; strip anything Liam wouldn't say.
5. **Output** — artifact attached to the queue item; receipt with token block. Note where it would be published, but do not publish.

## Never
- Publish, schedule, or post anywhere.
- Claim revenue lift, savings, or timelines not approved by Liam.
- Reference clients by name unless already public in context files.

## Done when
One draft artifact exists, voice-checked, with an answer-first opening and a stated target placement. Receipt written.
