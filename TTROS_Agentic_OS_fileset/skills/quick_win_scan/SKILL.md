# /quick-win-scan
> Revisit: if Quick-Win Scan pricing, scope, or report format changes. · Last touched: 2026-07-07

## Purpose
Turn one paid diagnostic conversation into a short, executive-feeling report on a single workflow or lead path. Scope is deliberately narrow — one workflow, not the business.

## Inputs
- Client entity page + intake notes/transcript from Liam's call. Refuse to start without both.

## Steps
1. **Confirm scope** — exactly one workflow, lead path, or bottleneck. If notes span several, ask Liam which one; do not widen scope.
2. **Map** — inputs, outputs, tools, delays, handoffs, data sensitivity, owner (the Audit lens from the AOA framework).
3. **Diagnose** — name the single highest-friction bottleneck; tag it to one ROI lever (effectiveness / efficiency / quality) with the reasoning visible.
4. **Prescribe** — one practical fix: off-the-shelf tool, workflow redesign, small automation, or a scoped build. Use effort-vs-impact framing. Label whether it's DIY, done-with-you, or done-for-you, with cost/effort range where known.
5. **Report** — short report artifact: current state, the bottleneck, why it matters, the fix, first step, and the natural upgrade path (Business Efficiency Assessment or a build) without hard-selling. Assumptions clearly labelled.
6. **Close** — receipt with token breakdown; update client entity page.

## Never
- Deliver 3–7 bottlenecks — that's the CA$750 assessment, not this.
- Quote specific ROI without evidence; unlabelled assumptions.
- Start implementation work inside the scan.
- Reuse another client's data or examples.

## Done when
One-page-ish report artifact exists covering one workflow with one tagged bottleneck and one recommended next step; delivered within the 48h target where practical; receipt + entity page updated.
